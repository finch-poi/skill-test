# 我看到的设计图上这个按钮的圆角是100%，找出为什么你得到的信息和我看到的信息存在分歧

原因找到了。**这是 DSL 导出机制的限制导致的分歧**，分三层解释：

---

### 分歧根本原因

**DSL 中的 `上传 Upload/按钮` 节点是 `type: INSTANCE`**，即一个组件库组件实例。Pixso MCP 的 `getNodeDSL` 对 INSTANCE 类型节点**不展开内部子节点**——它只记录实例外层容器的覆盖属性（overrides），组件内部的结构由组件主定义（master component）管理，不会写入 `pixDslNodes` 数组。

---

### 两个圆角各属于哪一层

| 圆角值 | 所属层级 | 说明 |
|--------|---------|------|
| `cornerRadius: 4` | **INSTANCE 外层容器** | DSL 能拿到的，是整个上传区域容器的圆角 |
| `100% / pill` | **组件内部的按钮元素** | 藏在组件 master 定义里，DSL **未展开**，MCP 拿不到 |

---

### 验证依据

```
"type": "INSTANCE"          ← 组件实例，非普通 FRAME
"internalOnly": false       ← 外层暴露
子节点查询结果: 0 个         ← 内部完全不展开
```

整个 DSL 中 `cornerRadius >= 20` 的节点只有 **1 个**（`下拉选择器 Select`，cornerRadius: 100），而上传按钮的内部结构完全缺失。

---

### 结论

你在设计图上看到的 100% 圆角是上传组件**内部**的按钮样式，属于 Pixso 组件库 master 层级的定义。MCP 返回的 DSL 只到实例外壳这一层（`cornerRadius: 4`），内部按钮的样式信息在这次 DSL 导出中**根本没有被包含**，所以两边数据天然存在这个盲区。
